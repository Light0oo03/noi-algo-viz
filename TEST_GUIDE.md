# NOI 算法可视化平台 - 栈和队列模块测试指南

## 测试环境

**开发服务器**：http://localhost:5177/
**测试路径**：http://localhost:5177/playground

## 测试前准备

1. 打开浏览器（推荐 Chrome/Edge/Firefox）
2. 访问 http://localhost:5177/playground
3. 打开浏览器开发者工具（F12）
4. 确保 Console 标签可见，用于监控错误

---

## 测试清单

### 一、栈算法测试（5个）

#### 1.1 有效的括号

**步骤**：
1. 侧边栏 → 栈 → 有效的括号
2. 点击"播放"按钮
3. 观察栈动画：括号依次入栈/弹出
4. 点击"暂停"按钮
5. 点击"单步"按钮（多次）
6. 点击"重置"按钮

**检查点**：
- [ ] 显示一个栈（垂直排列）
- [ ] top 指针在栈顶中间位置（"top ↓"）
- [ ] 括号依次入栈（绿色），匹配后弹出（红色）
- [ ] 代码行号高亮与动画同步
- [ ] 状态面板说明文字清晰（包含 emoji）
- [ ] 控制台无错误
- [ ] 播放/暂停/单步/重置功能正常

#### 1.2 最小栈

**步骤**：
1. 侧边栏 → 栈 → 最小栈
2. 点击"播放"按钮
3. 观察双栈动画

**检查点**：
- [ ] 显示两个栈（左：主栈，右：最小值栈）
- [ ] 左边栈的 top 指针在左侧（"top →"）
- [ ] 右边栈的 top 指针在右侧（"← top"）
- [ ] 两个栈同步更新
- [ ] push/pop/getMin 操作正确
- [ ] 代码高亮正确
- [ ] 控制台无错误

#### 1.3 单调栈（柱状图）

**步骤**：
1. 侧边栏 → 栈 → 单调栈（柱状图）
2. 点击"播放"按钮
3. 观察单调栈计算最大矩形面积

**检查点**：
- [ ] 显示单调栈（垂直排列）
- [ ] 当前比较元素高亮（蓝色）
- [ ] 弹出元素有弹出动画（红色）
- [ ] 计算最大矩形面积正确
- [ ] 说明文字显示当前面积计算
- [ ] 代码高亮与步骤同步
- [ ] 控制台无错误

#### 1.4 逆波兰表达式

**步骤**：
1. 侧边栏 → 栈 → 逆波兰表达式
2. 点击"播放"按钮
3. 观察栈计算表达式 ["2", "1", "+", "3", "*"]

**检查点**：
- [ ] 显示一个栈
- [ ] 操作数正确入栈（绿色）
- [ ] 运算符触发弹出和计算（蓝色）
- [ ] 最终结果正确显示（9）
- [ ] 说明文字显示当前操作
- [ ] 代码高亮正确
- [ ] 控制台无错误

#### 1.5 栈实现队列

**步骤**：
1. 侧边栏 → 栈 → 栈实现队列
2. 点击"播放"按钮
3. 观察两个栈实现队列操作

**检查点**：
- [ ] 显示两个栈（左：入栈，右：出栈）
- [ ] 两个栈的 top 指针位置合理（左右分布）
- [ ] enqueue 操作：元素加入入栈
- [ ] dequeue 操作：从出栈弹出（若空则从入栈转移）
- [ ] 说明文字描述队列操作

- [ ] 控制台无错误

### 三、通用功能测试

#### 3.1 菜单导航

**步骤**：
1. 连续切换不同算法：栈 → 队列 → 链表 → 图
2. 每次切换后点击播放
3. 观察数据结构是否正确刷新

**检查点**：
- [ ] 从栈切换到队列：无报错，数据正确
- [ ] 从队列切换到链表：无报错，数据正确
- [ ] 从链表切换到图：无报错，数据正确
- [ ] 每次切换后旧数据清空
- [ ] 控制台无错误

#### 3.2 播放控制

**步骤**：
1. 选择任意栈/队列算法
2. 测试所有控制按钮

**检查点**：
- [ ] 播放：自动逐步执行，状态实时更新
- [ ] 暂停：停止动画，保持当前状态
- [ ] 单步：每次点击执行一步，进度条更新
- [ ] 后退：回到上一步，状态正确回退
- [ ] 重置：回到初始状态，清空高亮
- [ ] 进度条：始终可见，拖拽可跳转

#### 3.3 代码显示

**步骤**：
1. 选择不同的栈/队列算法
2. 播放动画，观察代码高亮

**检查点**：
- [ ] CodeViewer 显示对应算法的代码
- [ ] 代码行号与动画步骤同步高亮（黄色背景）
- [ ] 语法高亮正确（关键字、字符串等）
- [ ] 代码可滚动查看

#### 3.4 状态面板

**步骤**：
1. 选择任意栈/队列算法
2. 播放动画，观察状态面板

**检查点**：
- [ ] 说明文字清晰（使用 emoji 和多行格式）
- [ ] 栈信息：栈 ID、栈深度、栈顶元素
- [ ] 队列信息：队列 ID、队列长度、front/rear 值
- [ ] 状态面板自动滚动（如内容过多）

#### 3.5 性能检查

**步骤**：
1. 打开浏览器开发者工具 → Performance 标签
2. 连续播放多个算法（各播放 3 次）
3. 观察性能指标

**检查点**：
- [ ] 无未捕获的异常（Console 标签）
- [ ] 帧率稳定（接近 60fps）
- [ ] 无内存泄漏（播放多次后内存不持续增长）
- [ ] 动画流畅，无明显卡顿
