# Totuma 对标差距追踪（noi-algo-viz）

更新时间：2026-02-25
对标站点：https://www.totuma.cn/algorithms/list/link-head-node.html

## 目标
用于持续跟踪“我方与对标站”的差距、实现状态、验收标准，避免遗漏。

## 差距清单（按优先级）

### P0 算法覆盖
- 状态：进行中
- 差距：
  - 图算法：缺 Floyd、Bellman-Ford、拓扑排序、关键路径。
  - 查找：缺 哈希查找（拉链/开放定址）、B 树查找等。
  - 排序：缺 希尔排序、基数排序、折半插入排序等。
- 验收：侧边栏可见、可播放、代码高亮同步、输入参数可配置。

### P0 持久化一致性
- 状态：进行中
- 差距：
  - 图/树已具备后端持久化。
  - 查找/排序目前仅本地持久化，尚未实现后端按账号同步。
- 验收：
  - 登录后优先读后端；后端无数据回退本地。
  - 修改后自动保存（本地+后端）。
  - 不同账号隔离，退出登录前尽力同步。

### P1 教学内容完整度
- 状态：未开始
- 差距：缺系统化的算法说明（思想、步骤、复杂度、适用场景、易错点）。
- 验收：每个算法页至少包含“简介 + 复杂度 + 关键步骤 + 示例输入输出”。

### P1 交互能力
- 状态：部分完成
- 差距：缺统一的“全屏/沉浸模式”、“递归栈专用视图”等高级交互。
- 验收：关键递归算法可切换栈视图；动画区支持全屏。

### P2 学习路径化
- 状态：未开始
- 差距：当前为功能菜单，不是章节式学习流。
- 验收：支持按主题/难度推荐学习顺序与已学进度。

## 本周执行顺序
1. 完成查找/排序后端持久化（P0）。
2. 继续补图算法（Floyd、拓扑排序优先）（P0）。
3. 补排序（希尔、基数）与查找（哈希）核心算法（P0）。
